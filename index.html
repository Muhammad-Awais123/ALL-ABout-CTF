Slide 02

What is CTF?

CTF stands for "Capture The Flag"

The name comes from a traditional outdoor game, adapted for the digital world

A cybersecurity competition with challenges

Participants solve puzzles and tasks to find hidden "flags"

A flag is usually a piece of text or code

Common format: flag{some_text_here}

A game that teaches real-world skills safely

Learn hacking and security concepts in a legal, controlled environment

Slide 03

Purpose of CTF

Hands-On Practice

Learn cybersecurity concepts by actually doing them, not just reading about them

Problem-Solving Skills

Develop critical thinking that helps in all areas of tech and security

Real-World Preparation

Practice scenarios that mirror actual security threats and vulnerabilities

Teamwork & Collaboration

Most CTFs are team-based — build communication and collaboration skills

Have Fun While Learning Ethical Hacking

The gamified format keeps you motivated and makes complex topics enjoyable to learn

Slide 07

What is Web CTF?

Focus Area

Finding and exploiting vulnerabilities in web applications — the most common attack surface on the internet

Simulated Attacks

SQL injection, XSS, CSRF, and more — these are the same flaws found in real websites

Where Flags Hide

Flags are often hidden in databases, source code comments, server files, or HTTP headers

The Goal

Think like an attacker to understand how websites can be broken — then learn how to fix them

Web challenges are usually the most beginner-friendly category in CTF competitions

Slide 08

Web Application Structure

Frontend

HTML, CSS, JavaScript — what users see

Backend

PHP, Python, Node.js — handles logic

Database

MySQL, SQLite — stores data & flags

Client → Server → Database

Communication

Browser talks to server using HTTP requests — GET to fetch data, POST to submit data

Cookies & Sessions

Small pieces of data stored in your browser — used to keep you logged in and track your activity on the site

Source Code

HTML/JS sent to your browser can be viewed directly — sometimes developers leave clues or flags in comments

Slide 09

How to Solve Web CTF Challenges

1

Explore the website

Click all buttons and links — note anything unusual

2

View page source code

Press Ctrl+U for hidden clues in HTML comments

3

Check robots.txt & sitemap

Look for hidden directories and files

4

Inspect cookies & headers

Flags can be hidden in response headers or cookie values

5

Test common vulnerabilities

SQL injection, XSS, file upload bypasses

6

Use developer tools

Press F12 to monitor network requests and responses

7

Think creatively

Sometimes the simplest solution works — don't overcomplicate things!

Slide 10

Tools Used for Web CTF

Burp Suite

Intercept and modify web traffic

Browser DevTools (F12)

Inspect elements, console, network

SQLmap

Automate SQL injection attacks

Dirbuster / Gobuster

Find hidden directories and files

curl / wget

Command-line tools for web requests

CyberChef

Data decoding and analysis in browser

Postman

Test and interact with APIs

Nmap

Scan for open ports and services
